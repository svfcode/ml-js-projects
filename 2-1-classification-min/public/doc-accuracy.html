Метрика accuracy в демо классификации

Эта страница относится к демо 2-1-classification-min и поясняет функцию accuracy() из app.js (строки 75–92).

Что такое accuracy

Accuracy — это доля правильных предсказаний модели в выборке:

accuracy = (количество правильных предсказаний) / (общее количество предсказаний)

В двуклассовой задаче (классы 0 и 1) каждое наблюдение либо предсказано верно, либо нет, поэтому accuracy — это просто процент попаданий.

Какие данные у нас есть

Точки на графике хранятся в массиве:

canvas.points = [ { x, y, label }, ... ]
  • x, y — координаты точки на плоскости;
  • label — настоящий класс точки: 0 или 1.

Модель логистической регрессии задаётся параметрами w₁, w₂, b внутри объекта model.

Шаги функции accuracy()

Упрощённая версия кода:

const pts = canvas.points;
const n = pts.length;
if (!n) return 0;
let c = 0;
for (const p of pts) {
  const z = w1 * p.x + w2 * p.y + b;
  const a = sigmoid(z);
  const pred = a >= 0.5 ? 1 : 0;
  if (pred === p.label) c++;
}
return c / n;

Разбор по шагам:

  • pts и n — берём все текущие точки и их количество. Если точек нет, accuracy считаем равной 0.
  • z = w₁·x + w₂·y + b — линейная комбинация признаков для точки p.
  • a = sigmoid(z) — модель выдаёт вероятность P(y=1 | x) для этой точки.
  • pred = a >= 0.5 ? 1 : 0 — мы выбираем класс по порогу 0.5:
    • если вероятность ≥ 0.5 → предсказываем класс 1;
    • иначе → класс 0.
  • if (pred === p.label) c++; — если предсказанный класс совпал с настоящей меткой точки, увеличиваем счётчик верных предсказаний.

В конце:

accuracy = c / n

Здесь c — количество попаданий, а n — количество всех точек, поэтому отношение c / n — это доля верных ответов. В интерфейсе демо это значение умножается на 100 и показывается в процентах.

Как линейная комбинация даёт метку класса

Для каждой точки p = (x₁, x₂) модель сначала считает линейную комбинацию признаков:

z = w₁·x₁ + w₂·x₂ + b
  • x₁, x₂ — признаки (координаты точки);
  • w₁, w₂ — веса (насколько важна каждая координата);
  • b — смещение (bias).

Затем эта линейная комбинация подаётся в сигмоиду:

P(y=1 | x) = σ(z) = 1 / (1 + e−z)

Получив вероятность класса 1, модель применяет порог 0.5, чтобы решить, к какому классу отнести точку:

const a = sigmoid(z); // P(y=1 | x)
const pred = a >= 0.5 ? 1 : 0;

Геометрически выражение w₁·x₁ + w₂·x₂ + b = 0 задаёт линию на плоскости, которая делит все точки на две полуплоскости:

  • там, где z > 0, модель склоняется к классу 1;
  • там, где z < 0, — к классу 0.

Именно эта линия рисуется в демо как оранжевая “граница решения”, а метрика accuracy проверяет, насколько часто такое правило попадает в реальные метки label.

Важно понимать, что accuracy полезна, когда классы сбалансированы. Если почти все точки принадлежат одному классу, высокая accuracy может означать всего лишь «всегда угадываем частый класс».
