Устройство

Как в демо устроены данные, шаг градиентного спуска, состояние и обновление интерфейса. Соответствует коду в public/ml.js, public/ui.js, public/logic.js, public/ux.js.

Архитектура скриптов

Файл Роль
ml.js Математика: синтетические данные, MSE, один шаг градиента. Без DOM.
ui.js Chart.js: графики данных и MSE, метрики на экране, состояние кнопок.
logic.js Состояние датасета и весов, история MSE, шаги runSteps, смена данных. Только LRML, без DOM.
ux.js DOM, события, скорость обучения из полей ввода, цикл «Обучать», вызовы LRLOGIC и LRUI.

Загрузка в index.html: CDN Chart.js, затем ml.js → ui.js → logic.js → ux.js.

Модель и потери

  • Предсказание: ŷ = w·x + b.
  • Параметры: w, b (хранятся в logic.js, объект LRLOGIC).
  • Данные: массив точек points в LRLOGIC, каждая точка { x, y }.
  • Функция потерь: MSE — среднеквадратичная ошибка по всем точкам:
    • для каждой точки ошибка: err = w·x + b − y;
    • MSE = (1/n) · Σ err².

Реализация: LRML.mse(points, w, b) в ml.js.

Состояние: logic.js и ux.js

В LRLOGIC (logic.js):

  • points — датасет.
  • w, b — текущие веса.
  • lossHistory — значения MSE после каждого шага (нижний график; история не обрезается).
  • step — счётчик шагов градиентного спуска (в метриках).

В ux.js (только интерфейс автозапуска):

  • running — идёт ли непрерывное обучение («Обучать»).
  • animId / lastTick — requestAnimationFrame и пауза ~55 ms между шагами.

Сброс истории MSE и счётчика шагов: LRLOGIC.resetTrainingOnly(). После правки точек вручную — LRLOGIC.resetAfterDatasetEdit() (то же самое).

Откуда берутся данные

Режим «Генерация»

  1. Берётся N из слайдера «Точек данных».
  2. Вызывается LRML.generateSyntheticDataset(n):
    • случайно задаются «истинные» trueW, trueB;
    • для каждой из n точек: x ∈ [−1, 1], шум ∈ [−0.35, 0.35], y = trueW·x + trueB + шум.
  3. Внутри LRLOGIC.generateSynthetic(n) вызываются LRML.generateSyntheticDataset, LRML.randomWeights() и resetTrainingOnly().
  4. В ux.js: generateData() вызывает LRLOGIC.generateSynthetic, затем syncMetrics() и drawAll().

Триггеры: переключение на режим «Генерация», кнопка «Сгенерировать», старт страницы (после загрузки).

Режим «Своими точками»

  • ЛКМ — LRLOGIC.tryAddPoint(x, y) (лимит LRLOGIC.MAX_MANUAL_POINTS, 200).
  • ПКМ — удалить ближайшую точку (радиус попадания в ui.js).
  • Переключение режима или «Очистить точки» — LRLOGIC.clearForManualMode().

Изменение датасета во время работы

onDatasetChanged() (добавление/удаление точки вручную):

  1. setRunning(false) — остановка автоматического обучения.
  2. LRLOGIC.resetAfterDatasetEdit() — чистая история MSE и счётчик шагов.
  3. Обновление метрик, перерисовка, updateModeUI().

Коэффициент η (learning rate) задаётся в интерфейсе и участвует в каждом вызове LRLOGIC.runSteps(n, lr). Реализация ползунка, числового поля, ограничений и того, как одно η действует на пачку шагов, разобрана на отдельной странице.

Пакет шагов и UI

LRLOGIC.runSteps(n, lr) (logic.js)

Условия выхода: нет точек или n < 1.

Цикл n раз:

  1. next = LRML.gradientStep(...) с переданным lr → записать w, b в LRLOGIC.
  2. step += 1.
  3. В lossHistory добавить LRML.mse(...).

ux.js: runTrainingSteps(n) вызывает getLearningRate() и LRLOGIC.runSteps(n, lr), затем syncMetrics() и drawAll().

Кнопки: «Один шаг» / «10 шагов» — runTrainingSteps(1) и runTrainingSteps(10).

Режим «Обучать»

  1. setRunning(true) — блокировка кнопок по правилам ui.js, старт цикла.
  2. На каждом кадре tick: если прошло ≥ 55 ms, вызывается runTrainingSteps(1).
  3. Снова планируется requestAnimationFrame(tick).
  4. «Пауза» — setRunning(false), отмена requestAnimationFrame.

При автоматическом режиме каждые ~55 ms выполняется один полный шаг (с перерисовкой после шага).

Отрисовка (ui.js + Chart.js)

Подключается Chart.js (CDN). initCharts создаёт смешанный график (scatter + линия) и график MSE; updateDataChart / updateLossChart обновляют данные. Подсказки при наведении показывают координаты точек и пару «шаг — MSE».

updateMetrics пишет в DOM текущие w, b, MSE, step.

Сводная цепочка «от клика до картинки»

  1. Пользователь нажимает «Один шаг» / «10 шагов» или идёт тик «Обучать».
  2. runTrainingSteps(n) передаёт η в LRLOGIC.runSteps; n раз обновляются w, b, дополняется lossHistory.
  3. LRUI.updateMetrics + LRUI.updateDataChart + LRUI.updateLossChart обновляют экран.

Загрузка страницы (демо)

  1. syncLrFromSlider() — согласовать ползунок и поле скорости обучения.
  2. generateData() — первый синтетический датасет и случайные w, b.
  3. updateModeUI() — видимость слайдера числа точек, подсказка ручного режима, подписи кнопок, доступность «Обучать» / шагов при пустых точках.
