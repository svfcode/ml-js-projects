Один шаг градиентного спуска (ml.js)

В демо минимизируется среднеквадратичная ошибка (MSE) по параметрам w и b линейной модели ŷ = w·x + b. Один шаг градиентного спуска реализован функцией LRML.gradientStep(points, w, b, lr) в файле public/ml.js.

MSE как функция w и b

На датасете из n точек (xi, yi):

  • Ошибка на точке: ei = w·xi + b − yi.
  • MSE: L(w, b) = (1/n) · Σi ei².

В коде mse как раз считает среднее квадратов err = w*p.x + b - p.y по всем точкам.

Частные производные L по w и по b

Цепное правило для одной точки: ∂(e²)/∂w = 2·e·x, ∂(e²)/∂b = 2·e. Усреднение по n даёт:

  • ∂L/∂w = (2/n) · Σi ei·xi
  • ∂L/∂b = (2/n) · Σi ei

Здесь ei в том же смысле, что err в цикле по точкам.

Пакетный градиент (full batch)

В демо используется полный градиент по всем точкам за один шаг (не мини-батч). В gradientStep сначала накапливаются суммы Σ(err·x) и Σ(err) без множителя 2/n, затем однократно умножаются на (2/n) — это и есть компоненты градиента ∇L = (∂L/∂w, ∂L/∂b).

Правило обновления

Градиентный спуск:

  • wновый = w − η · ∂L/∂w
  • bновый = b − η · ∂L/∂b

Параметр η — скорость обучения; в коде аргумент называется lr. Подробнее про выбор и ввод η см. страницу про learning rate.

Соответствие коду ml.js

Логика функции gradientStep строка в строку:

  1. Если точек нет (n === 0), вернуть исходные w, b без изменений.
  2. Для каждой точки: pred = w*x + b, err = pred - y, накопить dw += err * x, db += err.
  3. dw = (2/n) * dw, db = (2/n) * db.
  4. Вернуть { w: w - lr*dw, b: b - lr*db } — входные w, b не мутируются внутри функции; новые значения подставляются в состояние снаружи.

Вызов из logic.js

В LRLOGIC.runSteps в цикле вызывается LRML.gradientStep(points, w, b, lr); возвращённые веса записываются в замкнутые переменные модуля, затем увеличивается счётчик шага и в историю добавляется текущий LRML.mse(points, w, b).

Векторная форма (кратко)

Если собрать признаки в матрицу X с колонкой единиц для смещения, задача — минимизация ||Xθ − y||² по θ = (w, b)ᵀ. Градиент имеет вид (2/n)·Xᵀ(Xθ − y); покомпонентная запись совпадает с циклом по точкам в ml.js.

При изменении определения потерь или шага обновления правьте ml.js и эту страницу; обзор «Модель и потери» на Устройстве держите согласованным.
