Log-loss (бинарная кросс-энтропия)

В демо классификации обучение идёт на минимизацию средней log-loss — это то же самое, что средняя бинарная кросс-энтропия между истинной меткой y ∈ {0, 1} и предсказанной вероятностью класса 1: a = P(y=1 | x).

Модель в коде (public/ml.js)

Линейная часть (логит): z = w₁·x₁ + w₂·x₂ + b — функция linearPart(p, w1, w2, b) (строки ~58–60).

Вероятность: a = σ(z) — sigmoid(z) (строки ~14–17). Константа LOG_EPS (строка ~8) используется только внутри logLoss, чтобы не брать log(0).

Формула потерь на одной точке

Для одной пары (y, a):

ℓ(y, a) = −( y·log(a) + (1−y)·log(1−a) )

Если y = 1, остаётся −log(a): чем выше a, тем меньше штраф. Если y = 0, остаётся −log(1−a): чем ниже a, тем лучше.

По всему датасету из n точек в демо берётся среднее — см. деление на n в logLoss (строка ~73, цикл ~67–72).

Реализация LRML.logLoss

Фрагмент из public/ml.js (номера строк могут слегка сдвинуться при правках файла):

function logLoss(points, w1, w2, b) {
  const n = points.length;
  if (!n) return 0;
  let s = 0;
  for (const p of points) {
    const a = sigmoid(linearPart(p, w1, w2, b));
    const y = p.label;
    const t = Math.min(1 - LOG_EPS, Math.max(LOG_EPS, a));
    s += -(y * Math.log(t) + (1 - y) * Math.log(1 - t));
  }
  return s / n;
}

Здесь p.label — это \(y\), a — предсказание после сигмоиды. Переменная t — «зажатый» a в \([\text{LOG_EPS},\,1-\text{LOG_EPS}]\), чтобы логарифмы оставались конечными при экстремальных численных значениях.

Градиент и LRML.gradientStep

Для сигмоиды и кросс-энтропии производная по логиту сводится к простой разности err = a − y. Градиент среднего loss по весам (упрощённо): 

∂L/∂w₁ ≈ (1/n) · Σ (a − y)·x₁   (аналогично w₂ и x₂; для b — множитель 1)

Это ровно то, что делает цикл в gradientStep (строки ~99–104), затем усреднение k = 1/n и обновление с шагом lr (строки ~106–114).

В обучении вызывается не «голый» gradientStep для лога, а gradientStepTrace (с ~строки 126): он повторяет ту же математику и дополнительно собирает текст для панели «Лог последнего шага».

Связка logic.js → график и метрики

LRLOGIC.runSteps в public/logic.js в цикле вызывает LRML.gradientStepTrace, обновляет w1, w2, b, увеличивает step, пишет в lossHistory результат LRML.logLoss(...) после каждого шага — это значения на графике «Log-loss по шагам».

Текущие log-loss и accuracy в шапке панели считаются в ux.js через LRLOGIC.currentLogLoss() и LRLOGIC.currentAccuracy() (внутри — снова LRML.logLoss и LRML.accuracy).

Панель «Лог последнего шага»

Под кнопками на демо-странице выводится LRLOGIC.getStepLogText(): после хотя бы одного шага это текст последнего вызова gradientStepTrace; до первого шага — заглушка с текущими весами и logLoss.

Файлы: public/ml.js (модель, loss, шаг), public/logic.js (состояние, runSteps, getStepLogText), public/ux.js (обновление #logStepText в drawAll).
